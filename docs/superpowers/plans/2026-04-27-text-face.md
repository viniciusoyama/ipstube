# Text Face Implementation Plan

**Goal:** Add a "Text" face that shows one user-supplied character per panel across the 6 displays, in either fixed mode (first 6 chars) or sliding mode (left-to-right marquee with configurable interval and trailing padding). Foreground and background colors are user-configurable. The face can be toggled on/off and is switchable via MQTT.

**Architecture:** The face slots into the existing `IPSClock::TimeOrDate` enum as a 5th value (`TEXT = 4`). The existing `clockTaskFn` switch in `src/main.cpp` already dispatches on `getTimeOrDate()` and is also driven by MQTT, so the new face inherits MQTT switching for free. A new `TextAnimation` class mirrors `DigitalRainAnimation`'s `loop()` / `animate()` shape. Rendering uses per-panel `ChipSelect::setDigit(i)` because each panel shows a different character. A new `text` config namespace exposes settings via the existing `WSConfigHandler` plumbing, and a new `text.html` page is added to the web UI.

**Tech stack:** C++ / Arduino / PlatformIO, TFT_eSPI, ESPConfig, ESPAsyncWebServer, jQuery Mobile (existing web UI).

**Build verification:** ESP32 firmware. No host-side test runner. After each task, run `pio run -e ipstube` (default target). Build success = task done. Manual on-device verification at the end.

**Commit policy:** One commit per task. Plain commit messages. No `Co-authored-by:` line. No "🤖 Generated with..." footer.

**Slide formula reference:**
- `tape = text + (' ' repeated padding)` ; `L = tape.length()`
- At tick `t`, panel `i` (0 = leftmost, 5 = rightmost) shows `tape[((t - i) % L + L) % L]`
- Empty string in fixed mode → all panels show background only
- Slide interval is the dwell time per tick in milliseconds

```cpp
#define _TEXT_ANIMATION_H

#include <TFT_eSPI.h>
#include "ChipSelect.h"

class TFTs;

class TextAnimation {
public:

    // or fixed-mode content/colors/enabled state changed since last draw).
    bool loop();

    // Performs its own per-panel chip_select.setDigit(i) calls.

    void invalidate();

private:

    char tapeAt(const String& tape, int32_t idx) const;

    uint32_t lastDrawMs = 0;
    uint32_t tick = 0;
    bool dirty = true;
    String lastText;
    bool lastFixed = true;
    String lastFg;
    String lastBg;
};

#endif
```

**Step 2.2: Write `src/TextAnimation.cpp`**

```cpp
#include "TextAnimation.h"
#include "TFTs.h"
#include "IPSClock.h"

TextAnimation::TextAnimation() {}

uint16_t TextAnimation::parseHexColor(const String& s) {
    // Accepts "#rrggbb"; returns RGB565. Invalid input falls back to black.
    if (s.length() != 7 || s.charAt(0) != '#') return 0x0000;
    long v = strtol(s.c_str() + 1, nullptr, 16);
    uint8_t r = (v >> 16) & 0xFF;
    uint8_t g = (v >> 8) & 0xFF;
    uint8_t b = v & 0xFF;
    return ((r & 0xF8) << 8) | ((g & 0xFC) << 3) | (b >> 3);
}

char TextAnimation::tapeAt(const String& tape, int32_t idx) const {
    int32_t L = (int32_t)tape.length();
    if (L == 0) return ' ';
    int32_t m = ((idx % L) + L) % L;
    return tape.charAt(m);
}

void TextAnimation::invalidate() { dirty = true; }

bool TextAnimation::loop() {
    String text = IPSClock::getTextContent();
    bool fixed = IPSClock::getTextFixed();
    String fg = IPSClock::getTextFgColor();
    String bg = IPSClock::getTextBgColor();

    if (text != lastText || fixed != lastFixed || fg != lastFg || bg != lastBg) {
        lastText = text; lastFixed = fixed; lastFg = fg; lastBg = bg;
        dirty = true;
        tick = 0;  // restart slide cycle on config change
    }

    if (fixed) {
        if (dirty) { dirty = false; return true; }
        return false;
    }

    uint32_t interval = (uint32_t)IPSClock::getTextInterval().value;
    if (interval < 50) interval = 50;  // floor
    uint32_t now = millis();
    if (dirty || (now - lastDrawMs) >= interval) {
        if (!dirty) tick++;
        dirty = false;
        lastDrawMs = now;
        return true;
    }
    return false;
}

void TextAnimation::animate(TFTs& tfts) {
    String text = IPSClock::getTextContent();
    bool fixed = IPSClock::getTextFixed();
    uint8_t padding = IPSClock::getTextPadding();
    uint16_t fg565 = parseHexColor(IPSClock::getTextFgColor());
    uint16_t bg565 = parseHexColor(IPSClock::getTextBgColor());

    String tape;
    if (fixed) {
        tape = text;
    } else {
        tape = text;
        for (uint8_t k = 0; k < padding; k++) tape += ' ';
    }

    StaticSprite& sprite = tfts.getSprite();
    int16_t cx = sprite.width() / 2;
    int16_t cy = sprite.height() / 2;

    for (uint8_t i = 0; i < NUM_DIGITS; i++) {
        char ch;
        if (fixed) {
            // Panel 5 = leftmost (hours tens), panel 0 = rightmost (seconds ones).
            // User wants left-to-right reading order, so map char index 0 to panel 5.
            uint8_t charIdx = NUM_DIGITS - 1 - i;
            ch = (charIdx < text.length()) ? text.charAt(charIdx) : ' ';
        } else {
            // Sliding: at tick t, leftmost panel shows tape[t], next shows tape[t-1], ...
            uint8_t leftToRight = NUM_DIGITS - 1 - i;
            ch = tape.length() == 0 ? ' ' : tapeAt(tape, (int32_t)tick - (int32_t)leftToRight);
        }

        tfts.chip_select.setDigit(i);
        sprite.fillSprite(bg565);
        if (ch != ' ') {
            sprite.setTextDatum(MC_DATUM);
            sprite.setTextColor(fg565, bg565);
            sprite.setFreeFont(&FreeSansBold24pt7b);
            sprite.setTextSize(2);
            char buf[2] = { ch, 0 };
            sprite.drawString(buf, cx, cy);
        }
        sprite.pushSprite(0, 0);
    }
}
```

Note: `NUM_DIGITS` is 6 and is defined in `src/GLOBAL_DEFINES.h` (already included via `ChipSelect.h`). Panel index 0 corresponds to seconds-ones (rightmost) per the comment in `ChipSelect.h:7`, so we invert the mapping so the user-supplied text reads left-to-right in physical order.

`FreeSansBold24pt7b` is included by TFT_eSPI when `LOAD_GFXFF` is set, which is already in `platformio.ini:46`. `setTextSize(2)` scales it up to roughly fill the 135x240 panel.

```cpp
TextAnimation& TFTs::getTextAnimator() {
    static TextAnimation animator;
    return animator;
}

void TFTs::invalidateTextAnimation() {
    getTextAnimator().invalidate();
}

void TFTs::animateText() {
    TextAnimation& animator = getTextAnimator();
    if (!animator.loop()) {
        return;
    }

    claim();
    uint8_t saved = chip_select.getDigitMap();

    animator.animate(*this);

    chip_select.setDigitMap(saved, true);
    release();
}
```

Note: unlike `animateRain` which uses `setAll()` and one push, `animateText` does its own per-panel `setDigit(i)` inside `animator.animate()` so the saved/restored CS map covers everything.

## Notes / non-goals

- No new MQTT topic for `text_enabled` or `text_content`. Switching is reused via the existing `display` topic (TEXT = 4). If the user later wants direct MQTT control over the text string, that's a follow-up commit that adds a `text_content` topic to `src/mqttBroker.cpp` mirroring the `customDataTopic` handler at `src/mqttBroker.cpp:96`.
- No screensaver interaction with text mode. Text is treated as a face like TIME/DATE/WEATHER, so the existing screensaver pre-empts it.
- No font selection UI. One fixed font (`FreeSansBold24pt7b` scaled 2x) per the spec.
- Unicode/emoji not supported. The bundled GFX font is ASCII.
